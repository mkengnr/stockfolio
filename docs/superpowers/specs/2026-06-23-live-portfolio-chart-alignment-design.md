# 실시간 포트폴리오 차트 및 날짜 정렬 개선 설계

## 배경

현재 메인 대시보드와 공유 페이지의 포트폴리오 차트는 확정된 일별 스냅샷만 사용한다. 따라서 장중 상단 요약 카드와 보유 종목 표는 현재가를 보여주지만, 차트의 마지막 점은 직전 확정 종가에 머문다.

상단 평가금액 차트와 하단 일별손익 차트는 서로 다른 데이터 개수로 생성되는데도 `LogicalRange`를 그대로 동기화한다. 하단 차트에는 첫 이력 날짜의 손익 막대가 없으므로 논리 인덱스가 한 칸 짧고, 같은 인덱스가 서로 다른 날짜를 가리킨다. 이 때문에 상단 막대가 하단 막대보다 오른쪽으로 한 칸 이동해 보인다.

또한 상단 차트는 시간축을 숨겨 두어 마우스를 올려도 하단 차트와 같은 날짜 라벨이 나타나지 않는다. 공유 페이지에서만 평가금액과 원금 기준선 사이의 수익·손실 음영을 활성화하고 있어 메인 페이지와 시각 표현도 일치하지 않는다.

## 목표

- 메인 대시보드와 공유 페이지 차트의 마지막 점에 현재가 기준 평가 상태를 표시한다.
- 현재가 점은 확정 종가 점과 시각적으로 동일하게 표시한다.
- 장 종료 후 같은 날짜의 확정 스냅샷이 생기면 중복 없이 같은 위치에서 대체되게 한다.
- 상단과 하단 차트의 동일 날짜가 같은 X 좌표에 오도록 한다.
- 상단 차트에서도 기본 시간축 날짜와 호버 날짜 라벨을 표시한다.
- 공유 페이지에 적용된 원금 대비 수익·손실 음영을 메인 페이지에도 적용한다.
- 상단 요약의 당일손익과 하단 차트의 현재 날짜 막대가 일치하게 한다.
- 확정 스냅샷과 과거 운영 데이터는 변경하지 않는다.

## 비목표

- 장중 현재가를 `daily_snapshots`에 저장하지 않는다.
- 스냅샷 수집 주기나 종가 확정 규칙을 변경하지 않는다.
- TradingView 차트 라이브러리를 교체하지 않는다.
- 현재가 점에 별도 색상, 점선, 배지 또는 `잠정` 라벨을 추가하지 않는다.
- 기존 손익 계산식이나 시장별 당일손익 기준일을 다시 설계하지 않는다.

## 선택한 접근

### 프런트엔드 가상 현재가 점

기존 대시보드 응답의 `summary`, `groups`, `current_price_as_of`, `price_dates_by_market`를 이용해 렌더링 전용 현재가 점을 만든다. 이 점은 차트 데이터에만 병합하며 백엔드 이력과 데이터베이스에는 쓰지 않는다.

이 방식을 선택한 이유는 다음과 같다.

- 현재 응답에 차트에 필요한 현재 평가금액, 투자원금, 잔여원금, 평가손익, 당일손익이 이미 있다.
- 확정 이력의 의미를 유지하면서 화면만 현재 상태로 확장할 수 있다.
- 메인, 라벨 범위, 공개 공유 범위에 같은 차트 컴포넌트와 병합 함수를 재사용할 수 있다.
- 별도 실시간 차트 API나 중복 응답 필드를 추가하지 않아도 된다.

검토 후 제외한 대안은 다음과 같다.

1. 백엔드에서 현재가를 `history.rows`에 삽입: 구현 위치는 중앙화되지만 확정 이력과 장중 값의 의미가 섞인다.
2. 별도 실시간 차트 API 추가: 데이터 계약은 명확하지만 현재 응답과 계산을 중복하고 요청 수도 늘어난다.

## 데이터 계약과 병합 규칙

### 공통 현재가 입력

`PortfolioChart`는 기존 확정 이력과 별도로 렌더링 전용 현재가 입력을 받는다. 현재가 입력은 다음 개념을 포함한다.

- 차트 날짜
- 현재 선택 범위의 `DashboardSummary`
- 전체 화면에서 그룹 구성 막대를 만들 때 사용할 현재 출처 그룹 요약 목록

호출부가 선택 범위를 결정하고, 공통 차트는 현재가 입력을 확정 이력과 병합한다. 메인 대시보드, 라벨 대시보드, 공유 페이지가 같은 병합 경로를 사용한다.

### 현재 날짜 결정

- 메인, 라벨, 공유 대시보드는 `price_dates_by_market` 값 중 가장 최신 날짜를 차트의 현재가 날짜로 사용한다. 메인·라벨에서 시장별 날짜가 없을 때만 `current_price_as_of`로 대체한다.
- 날짜가 없거나 현재 평가금액이 없으면 현재가 점을 만들지 않고 확정 이력만 표시한다.
- 혼합 시장 포트폴리오의 한 점은 화면 상단의 시장별 현재가 기준과 함께 해석한다. 차트 자체에는 가장 최신 시장 날짜를 사용하며, 기존 `현재가 기준: 한국 … · 미국 …` 안내는 유지한다.

### 같은 날짜 병합

- 현재 날짜와 같은 확정 행이 없으면 현재가 행을 마지막에 추가한다.
- 같은 날짜의 확정 행이 있으면 해당 날짜의 렌더링 행을 현재 요약 값으로 교체한다.
- 그룹 종류와 그룹 ID를 포함한 키로 병합해 전체, 출처 그룹, 통합 그룹, 미분류 값이 섞이지 않게 한다.
- 병합 결과는 날짜순으로 다시 정렬하고 날짜·그룹별 중복을 허용하지 않는다.
- 현재가 점의 선, 막대, 색상, 굵기는 확정 종가 점과 동일하다.

### 현재 당일손익

과거 하단 막대는 기존처럼 인접한 확정 이력의 `total_profit_loss` 차이로 계산한다. 현재 날짜 막대만 선택 범위 요약의 `total_current_value_change`를 사용한다. 이 값은 이미 종목별 시장 현지 날짜와 직전 거래일을 기준으로 계산되므로 상단 당일손익 카드 및 보유 종목 합계와 일치한다.

현재 당일손익이 없으면 현재 날짜에는 값 없는 시간 점만 둔다. 임의로 0으로 표시하지 않는다.

## 차트 날짜 정렬

상단과 하단 차트가 같은 날짜 캘린더를 공유하도록 차트 데이터를 정규화한다.

- 평가금액·원금·현재가를 포함한 선택 범위의 날짜 목록을 하나 만든다.
- 하단 일별손익 데이터에도 모든 날짜를 넣는다.
- 손익을 계산할 수 없는 첫 날짜나 누락 날짜는 Lightweight Charts의 whitespace 데이터 `{ time }`로 넣는다.
- 상단 차트의 평가금액·원금 선은 값이 없는 날짜를 건너뛰므로, 상단 차트에도 하단과 동일한 전체 날짜를 whitespace 전용 선 시리즈(date spine)로 추가해 두 차트가 같은 날짜 수와 같은 논리 인덱스를 갖게 한다.
- 화면 이동과 확대·축소 동기화는 `subscribeVisibleLogicalRangeChange`와 `setVisibleLogicalRange`를 사용해 논리 인덱스로 전달한다. 날짜 범위(`setVisibleRange`)는 막대 경계로 반올림되어 두 차트가 미세하게 어긋나므로 동기화에는 쓰지 않는다(초기 위치 지정에만 사용). 공통 date spine 덕분에 논리 인덱스 동기화가 차트를 늘이지 않는다.
- `rightOffset`은 두 차트에 동일하게 유지한다.
- 숫자 자릿수 차이로 플롯 시작점이 달라지지 않도록 두 차트의 왼쪽 가격축에 동일한 충분한 `minimumWidth`를 적용한다.

이 규칙으로 첫 손익 막대가 없는 정상적인 경우뿐 아니라 중간 손익 값이 누락된 경우에도 이후 날짜가 밀리지 않는다.

## 시간축과 호버

- 상단 차트의 시간축을 활성화한다.
- 상단에 마우스를 올리면 Lightweight Charts 기본 크로스헤어와 날짜 라벨이 표시된다.
- 하단과 동일한 로케일·날짜 형식을 사용한다.
- 별도 DOM 툴팁은 만들지 않는다. 기본 시간축 동작을 사용해 키보드·마우스 동작과 차트 라이브러리 업데이트에 대한 유지보수 부담을 줄인다.

## 수익·손실 음영

공유 페이지에서 사용 중인 `GainLossBandPrimitive`를 메인 대시보드에서도 활성화한다.

- 평가금액이 선택한 원금 기준선보다 높으면 옅은 빨간색 음영을 표시한다.
- 평가금액이 기준선보다 낮으면 옅은 파란색 음영을 표시한다.
- 메인 페이지의 기본 기준은 기존과 같이 투자원금이다.
- 공유 페이지는 투자원금이 있으면 투자원금을 기본으로 하고, 없으면 기존 자동 선택 규칙에 따라 잔여원금을 사용한다.
- 사용자가 `투자원금`/`잔여원금` 토글을 변경하면 기준선과 음영이 함께 변경된다.
- 현재가 점도 음영 데이터에 포함해 음영이 마지막 점까지 이어지게 한다.

## 메인·라벨·공유 페이지 통합

### 메인 대시보드

- 선택한 전체/출처/통합/미분류 그룹 요약으로 현재 평가금액과 현재 당일손익을 만든다.
- 전체 선택 시 현재 출처 그룹 요약으로 마지막 그룹 구성 막대를 만든다.
- 수익·손실 음영을 항상 활성화한다.

### 라벨 대시보드

- 라벨 전용 대시보드의 요약과 기준일을 사용한다.
- 라벨 범위는 기존처럼 그룹 구성 막대를 표시하지 않는다.
- 현재가 점, 현재 당일손익, 날짜 정렬, 음영은 메인과 동일하게 적용한다.

### 공유 페이지

- 공유 대시보드의 선택된 전체/하위 출처 그룹 요약으로 현재가 점을 만든다.
- 전체 선택 시 공개 응답에 포함된 현재 출처 그룹 요약으로 그룹 구성 막대를 만든다.
- 현재가 점, 현재 당일손익, 날짜 정렬, 상단 날짜축, 음영을 메인과 같은 공통 차트 경로로 처리한다.
- 공개 응답에 내부 ID를 새로 노출하지 않는다.

## 안내 문구

현재가 점이 차트에 포함되므로 다음 문구는 더 이상 사실이 아니다.

> KRX 장중 현재가입니다. 차트는 직전 확정 종가까지 표시됩니다.

장중 경고는 다음처럼 변경한다.

> KRX 장중 현재가입니다.

US 등 다른 시장도 같은 형식을 사용한다. 공개 공유 응답의 경고 허용·정제 정규식과 관련 테스트도 새 문구에 맞춘다. 시장별 현재가 기준과 당일손익 기준 안내는 유지한다.

## 오류 및 결측 처리

- 현재 기준일이 없으면 확정 이력만 표시한다.
- 현재 평가금액이 없으면 현재가 점을 추가하지 않는다.
- 일부 그룹의 현재 평가금액이 없으면 해당 그룹의 현재 구성 막대만 생략하고 기존 데이터 품질 경고를 유지한다.
- 현재 당일손익이 없으면 현재 날짜의 하단 점은 whitespace로 처리한다.
- 확정 이력 자체가 없지만 현재 평가금액이 있으면 현재가 한 점만 표시한다. 음영은 두 점 이상일 때만 그린다.
- 빈 데이터 상태와 차트 로딩 실패 처리는 기존 동작을 유지한다.

## 테스트 전략

### 순수 데이터 함수

- 현재가 점을 마지막 날짜에 추가한다.
- 같은 날짜의 확정 행을 현재가 행으로 교체한다.
- 그룹 종류·ID별로 독립 병합한다.
- 기준일 또는 현재 평가금액이 없으면 확정 이력을 유지한다.
- 현재 당일손익은 `total_current_value_change`를 사용한다.
- 모든 날짜가 하단 데이터에 존재하고 첫 날짜·누락 값은 whitespace가 된다.
- 현재가 점까지 gain/loss band 데이터가 이어진다.

### 컴포넌트 테스트

- 상단 시간축이 보이도록 생성 옵션을 전달한다.
- 상·하단 차트가 동일한 날짜 캘린더와 오른쪽 여백을 사용한다.
- 메인 대시보드가 음영을 활성화한다.
- 투자원금/잔여원금 토글에 따라 기준선과 음영 입력이 함께 바뀐다.
- 메인, 라벨, 공유 호출부가 올바른 선택 범위 요약과 현재 날짜를 전달한다.
- 공유 페이지의 전체/하위 그룹 필터가 현재가 점과 하단 당일손익도 함께 전환한다.

### 백엔드 테스트

- 장중 경고가 `시장 장중 현재가입니다.` 형식으로 반환된다.
- 공개 공유 경고 정제가 새 형식은 유지하고 허용되지 않은 내부 경고는 계속 제거한다.

### 브라우저 검증

로그인된 인앱 브라우저를 사용해 다음을 확인한다.

- 메인 전체 및 각 그룹에서 마지막 평가금액이 현재 요약 카드와 일치한다.
- 하단 마지막 막대가 상단 당일손익 카드와 일치한다.
- 상단과 하단의 같은 날짜가 같은 X 좌표에 있다.
- 상단 호버 날짜가 표시된다.
- 투자원금/잔여원금 전환 시 선과 음영이 함께 변한다.
- 공유 페이지의 전체 및 하위 그룹에서도 같은 동작을 한다.
- 1개월, 3개월, 6개월, 1년, 전체 기간에서 마지막 현재가 점이 보인다.
- PC와 모바일 폭에서 차트가 잘리거나 겹치지 않는다.

## 완료 조건

- 메인과 공유 페이지 차트의 마지막 평가금액이 같은 화면의 현재 평가금액과 일치한다.
- 현재 날짜의 하단 막대가 같은 화면의 당일손익과 일치하거나, 당일손익이 없을 때는 표시되지 않는다.
- 상단과 하단의 날짜가 시각적으로 정렬된다.
- 상단 차트에서 날짜축과 호버 날짜를 확인할 수 있다.
- 메인과 공유 페이지 모두 선택 원금 기준에 따른 수익·손실 음영을 표시한다.
- 장중 안내가 차트 동작과 모순되지 않는다.
- 스냅샷·과거 운영 데이터에 쓰기가 발생하지 않는다.
- 관련 프런트엔드·백엔드 테스트와 빌드가 통과하고 인앱 브라우저 검증이 완료된다.
