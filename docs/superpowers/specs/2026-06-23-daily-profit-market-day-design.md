# 시장별 거래일 기준 당일손익 설계

## 목표

대시보드의 `전일대비`를 HTS 계좌 화면에서 일반적으로 쓰는 `당일손익`으로 변경한다. 단순히 문구만 바꾸지 않고, 아직 해당 시장의 오늘 시세가 없는 경우에는 그 시장의 당일손익 기여분을 0으로 처리한다.

저장된 일별 스냅샷과 종가 수집 일정은 변경하지 않는다. 변경 범위는 대시보드 조회 시 계산과 표시뿐이다.

## 용어와 계산 기준

- `당일손익`: 해당 시장의 오늘 현재 평가금액에서 직전 거래일 종가 기준 평가금액을 뺀 금액이다.
- KRX의 오늘은 `Asia/Seoul`, US의 오늘은 `America/New_York` 날짜로 판단한다.
- 종목의 `current_price_date`가 해당 시장의 오늘과 같을 때만 당일손익을 계산한다.
- `current_price_date`가 해당 시장의 오늘보다 이전이면 아직 오늘 시세가 없는 것으로 보고 당일손익을 0으로 반환한다.
- 현재가 또는 기준 종가를 구할 수 없으면 기존처럼 `null`을 유지한다. 데이터 부재를 실제 0원과 혼동하지 않는다.
- 전체와 그룹 당일손익은 화면에 포함된 종목별 당일손익의 합이다.

예를 들어 한국시간 2026-06-23 08:00에는 KRX의 시장 날짜가 6월 23일이므로 6월 22일 현재가는 이전 날짜로 판정되어 KRX 기여분은 0이다. 같은 순간 뉴욕은 6월 22일이므로 US의 6월 22일 현재가는 당일 시세이며, US 종목은 6월 18일 종가와 비교한 손익을 기여한다.

## 백엔드 설계

시장 세션 서비스에 시장별 현지 날짜를 반환하거나 현재가 날짜가 시장의 오늘인지 판정하는 순수 함수를 둔다. 대시보드 응답 생성 함수는 테스트 가능한 `now` 값을 받아 종목별 당일손익 계산에 전달한다.

응답에는 `daily_change_active_by_market: dict[str, bool]`을 추가한다. 프론트엔드는 브라우저 시각으로 다시 판단하지 않고 이 값을 사용해 시장별 당일손익 기준 문구를 만든다. 기존 금액 필드명(`current_value_change`)은 호환성을 위해 유지한다.

종목별 계산 순서는 다음과 같다.

1. 현재 평가금액 또는 현재가 날짜가 없으면 `null`을 반환한다.
2. 현재가 날짜가 시장의 오늘보다 이전이면 0을 반환한다.
3. 현재가 날짜가 시장의 오늘보다 미래이면 신뢰할 수 없는 데이터이므로 `null`을 반환하고 `현재가 기준일이 시장 날짜보다 미래입니다` 경고를 추가한다.
4. 현재가 날짜가 시장의 오늘이면 그 날짜보다 앞선 가장 최근 스냅샷 종가와 비교한다.
5. 표시 통화로 환산한 종목별 값을 합산해 그룹과 전체 당일손익 및 비율을 만든다.

## 화면 설계

다음 라벨을 모두 `당일손익`으로 통일한다.

- 전체 수익현황 요약 카드
- 그룹별 수익현황 열
- 보유 종목 열과 합계
- 공유 대시보드의 같은 화면

상단 기준 문구는 단순한 `비교 기준(직전 거래일)` 대신 시장 상태를 포함한 `당일손익 기준`으로 표시한다.

- 활성 시장: `미국 6/22 vs 6/18`
- 아직 오늘 시세가 없는 시장: `한국 당일 시세 없음`

`당일 시세 없음`은 장 시작 전뿐 아니라 주말과 휴장일에도 정확한 표현이다.

## 오류 처리와 경계 조건

- 서로 다른 종목의 현재가 날짜가 다르면 기존 경고를 유지한다.
- 환율 또는 현재가가 없어 계산할 수 없는 종목은 0으로 강제하지 않고 `null`과 기존 경고를 유지한다.
- 당일 거래로 보유 수량이 바뀐 경우에도 기존 원장 기반 현재 수량과 직전 종가를 사용한다. 입출금·실현손익 처리 방식은 이번 범위에서 변경하지 않는다.
- 특별 지연 폐장일 설정과 스냅샷 확정 정책은 기존 로직을 그대로 사용한다.

## 테스트

백엔드 단위 테스트로 다음을 검증한다.

- 한국시간 장 시작 전 KRX 종목 당일손익은 0
- 같은 시각 US 종목은 뉴욕 현지 날짜가 현재가 날짜와 같으면 직전 종가 대비 손익 계산
- KRX와 US 혼합 계좌는 두 시장의 기여분을 합산
- 현재가 누락과 미래 날짜는 0이 아니라 `null`
- 그룹 합계와 전체 합계는 종목별 당일손익 합과 일치

프론트엔드 테스트로 요약 카드, 그룹 표, 종목 표의 라벨과 상단 시장별 기준 문구를 검증한다.

## 제외 범위

- DB 마이그레이션
- 스냅샷 수집 시각 또는 저장 방식 변경
- 과거 스냅샷 재작성
- 별도의 KRX/US 당일손익 카드 추가
- 내부 API 필드명 변경
