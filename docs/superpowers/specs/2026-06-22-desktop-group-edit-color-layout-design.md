# 데스크톱 그룹 수정 색상 배치 수정 설계

작성일: 2026-06-22
브랜치: `label-filter-and-group-color`

## 문제

그룹 카드는 데스크톱에서 2~3열 그리드로 배치된다. 인라인 편집 폼은 카드 너비가 좁아진 뒤에도 `sm` 이상에서 이름, 설명, 색상을 3열로 배치한다. 색상 프리셋은 자체 최소 너비를 가지므로 편집 카드 밖으로 넘쳐 다음 카드 위에 표시된다. 모바일에서는 3열 규칙이 적용되지 않아 정상이다.

## 결정

인라인 편집 카드의 이름, 설명, 색상 입력을 모든 화면 크기에서 세로로 배치한다. 순서는 `그룹 이름 수정 → 설명 수정 → 그룹 색상 수정`이다.

- 새 그룹 생성 폼의 넓은 데스크톱 레이아웃은 변경하지 않는다.
- 그룹 목록의 2~3열 카드 그리드는 변경하지 않는다.
- 색상 프리셋의 크기와 동작은 변경하지 않는다.
- 모바일의 기존 세로 배치는 그대로 유지한다.

전체 행 확장이나 컨테이너 쿼리도 가능하지만, 이번 문제에는 레이아웃 복잡도와 카드 재배치를 늘린다. 편집 카드 내부만 세로 배치하는 것이 가장 작고 안정적인 수정이다.

## 구현

`GroupCard` 편집 폼의 입력 래퍼에서 데스크톱 3열 클래스를 제거하고 `flex flex-col gap-4`를 사용한다. API, 상태, 저장 및 오류 처리 흐름은 바꾸지 않는다.

## 검증

- 컴포넌트 테스트에서 편집 폼 입력 래퍼가 세로 레이아웃이고 데스크톱 3열 클래스를 사용하지 않는지 확인한다.
- 기존 `GroupManager` 테스트 전체를 실행한다.
- 데스크톱 화면에서 색상 프리셋이 편집 카드 경계를 넘지 않는지 확인한다.
- 모바일 화면에서 입력 순서와 저장/취소 동작이 유지되는지 확인한다.

## 비범위

- 카드 전체 폭 확장
- 새 그룹 생성 폼 재설계
- 색상 팔레트 스타일 변경
- 다른 그룹 관리 기능 리팩터링
